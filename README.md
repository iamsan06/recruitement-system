# AI Recruitement System: Retrieval-Augmented Resume Screening Platform

An AI-powered recruitment intelligence platform that automates resume screening, candidate ranking, and job matching using retrieval-augmented techniques and large language models.

Instead of relying solely on keyword filtering, the system combines semantic retrieval, structured information extraction, dynamic scoring, and evidence-based reasoning to identify the most relevant candidates for a role.

The platform is designed to assist recruiters by producing explainable candidate rankings supported by retrieved resume evidence.

---

## Key Features

- Resume ingestion through PDF uploads
- Automated candidate profile extraction
- LLM-based job requirement analysis
- Hierarchical resume chunking
- Semantic retrieval using vector embeddings
- Dynamic multi-factor candidate scoring
- Evidence-backed recruiter explanations
- Skill gap analysis
- Candidate deduplication
- Persistent storage with PostgreSQL and Supabase
- Interactive Gradio interface
- Deployment on Hugging Face Spaces

## Candidate Intelligence Pipeline

### Resume Parsing

Uploaded resumes are converted from PDF documents into raw text using PyMuPDF.

### Candidate Information Extraction

The system uses Groq-hosted Llama models to extract structured information including:

- Personal information
- Technical skills
- Education history
- Work experience
- Projects
- Certifications
- Years of experience

### Hierarchical Chunking

Resumes are segmented into meaningful sections such as:

- Experience
- Projects
- Skills
- Education
- Certifications

Large sections are further subdivided to improve retrieval quality.

### Embedding Generation

Each chunk is embedded using:

```text
BAAI/bge-small-en-v1.5
```

Embeddings are normalized and stored within pgvector for efficient semantic search.

---

## Candidate Matching Strategy

Candidate ranking combines multiple signals.

### Semantic Relevance

Measures similarity between job descriptions and retrieved resume chunks.

### Skill Alignment

Calculates overlap between extracted candidate skills and job requirements.

### Experience Match

Evaluates whether candidate experience satisfies role expectations.

### Certification Match

Measures alignment between required and available certifications.

### Dynamic Weighting

Weights are automatically adjusted based on the information available within a job posting.

### Hard Filtering

Candidates with insufficient semantic relevance or poor skill alignment are removed before final ranking.

---

## Explainability

For every recommended candidate, the system generates recruiter-facing explanations describing:

- Why the candidate is relevant
- Matching technical skills
- Relevant experience and projects
- Missing qualifications
- Evidence supporting the recommendation

The language model is explicitly instructed to use only retrieved resume evidence and avoid unsupported assumptions.

## Deployment

The application is deployed on Hugging Face Spaces using Gradio, while FastAPI powers the backend services. Supabase provides PostgreSQL infrastructure, vector search capabilities through pgvector, and cloud storage for uploaded resumes.

---

## Limitations

- Resume formatting quality affects extraction performance.
- LLM outputs may occasionally require recruiter verification.
- Embedding quality depends on the selected embedding model.
- The platform is intended to assist human decision-making and should not be used as the sole basis for hiring decisions.

---

## Design Considerations

For simplicity, candidate metadata and chunk representations are consolidated within a single retrieval-oriented data model. In a production environment, these entities would typically be normalized into separate candidate and resume chunk tables.

The current design prioritizes simplicity, retrieval efficiency, and rapid deployment while maintaining explainable candidate recommendations.
